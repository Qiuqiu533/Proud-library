## v1.1 — AI書評品質管理基盤（2026-07-12〜13）

- **Version:** v1.1.0
- **Release Date:** 2026-07-13
- **Status:** Production

### 1. 概要

これまでのProud LibraryのAI書評機能は「ISBNから書誌情報を取得し、AIに生成させてDBに保存する」という一方向の流れだった。今回のリリースでは、この流れに**品質を検査・可視化・改善するループ**を組み込み、以下のサイクルとして完成させた。

```
ISBN → 書誌情報取得 → AI生成 → Validation Gate → 品質ティア判定
  → 要確認レビュー一覧 → 管理者選択 → 再生成 → 再評価
```

あわせて、ジャンル分類の精度改善・データ品質の可視化にも取り組んだ。今回のリリースは新機能の追加よりも**品質向上・運用性向上**が主軸である。

### 2. 新機能

| 機能 | 内容 |
|---|---|
| **AI書評 Validation Gate** | 生成された書評本文が、登録ジャンルと矛盾する断定表現（例:「ミステリ小説として」なのに実際はエッセイ）を含んでいないか、本文の長さが極端でないかを生成後に機械検査する。矛盾が見つかった場合はDB保存前に破棄する |
| **総合品質ティア（High/Medium/Low）** | AIの自己申告値であるconfidenceと、Validation結果（警告件数）を組み合わせた3段階の品質判定をDB保存。confidence単独では拾えない「確信度は高いが内容に懸念があるレビュー」も検出できる |
| **要確認レビュー一覧・選択再生成UI** | 品質ティアがMedium/Lowの本を管理画面に一覧表示（タイトル・confidence・判定理由付き）。チェックボックスで選択した分だけ、概算コストを確認したうえで再生成できる |
| **管理ダッシュボードの品質指標** | AI書評生成率・平均confidence・品質ティア内訳・要確認件数・NDC取得率・NDC未対応コード件数・ジャンル監査不一致件数をダッシュボード上に可視化 |

### 3. 改善点

| 項目 | 内容 |
|---|---|
| **KEYWORD_GENRE拡充** | NDC（日本十進分類法）が取得できない本でも、タイトルのキーワード（哲学・思想／経済・政治・法律・社会問題など）から分類できるよう補助ルールを強化。外部データソース（CiNii Books・版元ドットコム）は実測検証の結果、NDC取得用途には使えないと判断し不採用とした |
| **NDC778.8マッピング修正** | 特撮・アニメのテレビ絵本シリーズ（NDC778.8）が、演劇・映画系NDC（770〜779番台）の一括マッピングに巻き込まれ「エッセイ・評論」に誤分類されていた問題を修正。778.8のみ「絵本・児童書」へ個別マッピングし、ジャンル監査の誤検出を解消（監査不一致12件→9件） |
| **ジャンル監査精度向上** | 上記修正により、`genre-audit`（NDCと現在のジャンル設定の不一致検出）の精度が向上 |
| **キャッシュ更新運用の徹底** | `static/app.js`変更時の`STATIC_VERSION`/Service Worker `CACHE_NAME`更新を確実に行うプロセスを再徹底 |

### 4. 互換性・移行

- **DBマイグレーション: あり**（`genre_books.ai_quality_tier`カラム追加。デプロイ時に自動実行、既存データへの影響なし・後方互換）
- **API互換性: 維持**（既存エンドポイントの破壊的変更なし。新規エンドポイント追加のみ）
- **既存データの移行: 不要**（新カラムは新規生成・再生成分から順次埋まる。既存のAI書評は`ai_quality_tier`が未設定のままでも動作に支障なし）
- **影響範囲: 管理画面機能の追加が中心**。住民向け画面・API（蔵書検索・お気に入り・貸出等）への変更はなし

### 5. 品質・検証結果

- 全テスト: **252/252 PASS**（回帰テスト含む）
- CI: 全コミットで成功
- 本番デプロイ: Render.comへ全コミット反映確認済み
- **20冊での品質ティア実証**: 古典・翻訳文学・詩歌・評論・ジャンル境界作品を意図的に選定して再生成 → High 17件／Medium 3件／Low 0件。Medium判定は全てconfidence 60〜74のグレーゾーン境界に集中し、想定通りの分布
- **風姿花伝の再検証**: 過去にハルシネーション事故（能楽の古典を「ミステリ小説」として誤生成）を起こした対象を再生成し、Validation Gate導入後はHigh判定で正確な内容を確認。対策が実データで機能することを確認できた
- **実機確認**: 要確認レビュー一覧・選択・概算コスト表示・再生成ボタンの一連の動作を、実際のブラウザ・本番管理画面上で確認済み

### 6. 既知の事項

- **品質スコア推移（時系列トレンド）は未実装**: 品質ティア・confidenceの履歴を時系列で保存するテーブルがまだ存在しないため、ダッシュボードでは「現在の状態」のみを表示している。次期対応候補
- **「魔法戦隊マジレンジャー」のジャンル不一致**: NDC778.8マッピング修正により正しい提案（絵本・児童書）に変わったが、DB上の現在のジャンルは古いまま（ファンタジー・SF）で、ジャンル監査が引き続き不一致として検出している。監査は検出のみ・自動修正しない設計方針のため、意図的に未修正のまま残している

### 7. 今後のロードマップ

優先順位の高い順：

1. 品質スコア履歴・トレンドの実装（時系列テーブル追加）
2. データ品質ダッシュボードのさらなる可視化
3. ジャンル分類の継続的な改善（NDC 770番台の残りの精査等）
4. AI書評再生成の運用改善（バッチ分割・進捗ログ・実行履歴の永続化）
5. NDC補完のための代替データソース調査（CiNii Books・版元ドットコムは不採用と判断済み。大学図書館OPAC等は費用対効果が低いため保留）

### 成果サマリー

今回のリリースにより、AI書評は「生成」だけでなく、「品質判定・管理者レビュー・選択再生成」まで含めた運用サイクルを実現した。また、ジャンル分類品質やAI品質を管理画面から継続的に監視できるようになり、Proud Libraryの運用品質が大きく向上した。
