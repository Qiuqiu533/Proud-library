## 1. システム概要

```
住民・管理者（ブラウザ）
        ↓
  Render.com（サーバー・無料プラン）
        ↓
  PostgreSQL（Neon・無料プラン）
```

| 項目 | 内容 |
|------|------|
| サイトURL | https://proud-library.onrender.com |
| サーバー管理 | https://dashboard.render.com |
| コード管理 | https://github.com/Qiuqiu533/Proud-library |
| データベース | PostgreSQL（Neon）無料・無期限プラン |

### スリープについて
Renderの無料プランは約15分アクセスがないとスリープします。次のアクセス時に起動まで最大50秒かかります。4分ごとの自動pingを実装済みのため、誰かがアプリを開いている間はスリープしません。

## 2. 管理者メニューへのアクセス

### ログイン手順

1. 画面右上の「🏛️ 管理者メニュー」ボタンをクリック
2. ログイン画面で以下を入力：
   - **お名前**：チャットで表示される名前（例：田中、委員長）
   - **パスワード**：管理者パスワード（理事会メンバー共通）または理事会パスワード
3. 「ログイン」をクリック

### ロールの違い

| ロール | できること |
|--------|-----------|
| **管理者（ADMIN）** | 全機能＋住民アカウント管理・設定変更 |
| **理事会（BOARD）** | お知らせ・新着・課題・リクエスト・チャット等の運営機能 |

## 3. ダッシュボード

管理者メニューを開くと最初にダッシュボードが表示されます。

| カード | 内容 |
|--------|------|
| 未処理リクエスト | 住民からの購入リクエスト件数 |
| 未読フィードバック | 住民からの意見・要望件数 |
| 未解決の課題 | 課題管理の未クローズ件数 |
| 蔵書統計 | 総蔵書数・ジャンル分布 |
| 新着登録 | 直近の新着本 |

カードをクリックすると該当タブに直接移動します。

**クイックアクション**（ダッシュボード下部）

- 📢 お知らせを投稿
- 📥 新着本を登録
- 📝 書評を入力
- 📋 課題を追加
- 💬 チャットを開く
- ⚙️ 設定

## 4. お知らせを投稿する

「📢 お知らせ」タブ

1. タイトルと本文を入力
2. **重要度**を選択（通常 / 重要 / 緊急）
3. **固定表示**をONにすると一覧の最上部に固定されます
4. 「投稿する」をクリック

投稿されたお知らせは住民の「📢 お知らせ」タブに表示されます。未読バッジが表示されるので見落とし防止になります。

---

## 5. 新着本を登録する

「📥 新着登録」タブ

### ISBNで登録（推奨）

1. 本の裏表紙に記載されているISBN（13桁）を入力
2. 「検索」ボタンを押すと書名・著者が自動入力されます
3. 内容を確認して「登録」をクリック

### 手動で登録

ISBNがわからない場合は手動でタイトル・著者を入力できます。

### 登録後

「🆕 新着」タブに表示され、住民に通知されます。  
新着バッジ（🆕）がタブに表示されます。

---

## 6. 書評・内容紹介を入力する

「📝 書評入力」タブ

本のISBNまたはタイトルで検索し、内容紹介・書評文を入力します。  
入力した内容は書籍詳細モーダルの「内容紹介」として住民に表示されます。

## 7. リクエストを管理する

「📬 リクエスト」タブ

住民から届いた購入リクエストの一覧です。

### ステータス管理

| ステータス | 意味 |
|-----------|------|
| 未対応 | 確認待ち |
| 検討中 | 理事会で検討している |
| 購入済み | 蔵書に追加した |
| 見送り | 今回は購入しない |

ステータスを変更することで住民側にも反映されます。

### フィードバックの確認

「フィードバック」サブタブで住民からの意見・要望も確認できます。

### CSVエクスポート

「📥 CSV」ボタンから、リクエスト・ご要望の一覧をCSVでダウンロードできます。

## 13. 受賞作DBを管理する

「🏆 受賞作DB」タブ（管理者のみ）

直木賞・本屋大賞などの受賞作データを管理します。

> **重要：** データは公式サイト等で必ず確認してから追加してください。誤ったデータを追加すると住民に誤情報が伝わります。

### 受賞作の追加

1. 賞名・年・ISBN・作品名・著者を入力
2. ISBNを入力すると書名が自動補完されます
3. 「追加」をクリック

一覧から該当行の「編集」「削除」ボタンで操作できます。

## 15. 住民アカウントの管理

「👥 管理者設定」タブ（管理者のみ）

### できること

| 操作 | 説明 |
|------|------|
| アカウント一覧 | 登録住民の部屋番号・メール一覧 |
| パスワードリセット | 住民がパスワードを忘れた場合に初期化 |
| アカウント削除 | 退去した住民のアカウントを削除 |
| アカウント追加 | 管理者側から住民アカウントを作成 |

> **注意：** 削除したアカウントは復元できません。退去済みの部屋番号のみ削除してください。

### 招待コード管理（🎟️ 招待コード）

住民登録時に必要な招待コードを発行できます。発行枚数・メモ・有効期限（任意）を指定でき、期限を過ぎたコードは自動的に使用不可になります。`INVITE_REQUIRED=1` をRender環境変数に設定すると、招待コードなしでの登録を拒否します。

### QRコードポスター生成（🖨️ QRポスター）

掲示板・エントランスに貼るための、アプリURLへのQRコード付きポスター（PNG）を生成できます。見出し・サブ文言を入力して「ポスターを生成する」→「PNGでダウンロード」で印刷用画像が手に入ります。

また、各本の詳細画面（管理者ログイン中のみ表示）から「🏷️ この本のQRコードシールを表示」を選ぶと、その本専用のQRコードが生成されます。棚に貼っておくと、住民がスマホで読み取るだけで直接その本の詳細ページを開けます（棚卸し・蔵書整理時にも活用できます）。

## 17. サーバー・DB管理

### Render.com の確認

- ログイン：https://dashboard.render.com
- 「proud-library」サービスのログ・デプロイ状況を確認

### デプロイの仕組み

GitHubのmainブランチにコードがpushされると自動的にデプロイが始まります。デプロイ中（約2〜3分）はサイトが一時的に応答しないことがあります。

### Neon DBの確認

- ログイン：https://console.neon.tech
- 使用量・テーブル内容を確認できます

### データベースのバックアップ・リストア

**バックアップの仕組み**

- `.github/workflows/backup.yml` により毎日 JST 05:00 に自動でDBの全内容（`pg_dump`）を取得し、GitHub Actionsの「Artifacts」に30日間保存しています。
- 手動でバックアップを取りたい場合：GitHubリポジトリの「Actions」タブ →「DB日次バックアップ」→「Run workflow」で即座に実行できます。

**バックアップの確認・ダウンロード方法**

1. GitHubリポジトリ → 「Actions」タブ → 「DB日次バックアップ」を選択
2. 実行履歴一覧から復元したい日付の実行を選ぶ
3. ページ下部の「Artifacts」から `db-backup-XXX` をダウンロード（zip形式、中に `.sql` ファイルが1つ）

**リストア手順（緊急時・データ破損時）**

⚠️ リストアは既存のデータを上書きする破壊的操作です。必ず事前に理事会・開発者間で実施の合意を取ってから行ってください。

1. 上記の手順でバックアップの `.sql` ファイルをダウンロードし、展開する
2. Neonダッシュボード（https://console.neon.tech ）から対象プロジェクトの接続文字列（`DATABASE_URL`）を確認する
3. ローカルPCまたは信頼できる端末で以下を実行する（`psql` コマンドが必要）：
   ```bash
   psql "<Neonの接続文字列>" < proud_library_backup_YYYYMMDD_HHMMSS.sql
   ```
4. リストア完了後、アプリ（https://proud-library.onrender.com ）にアクセスし、蔵書・お知らせ・イベントなどが正しく表示されるか確認する
5. Renderのサービスを再起動する（DB接続プールをリフレッシュするため）

**Neon側の短期バックアップ（補助）**

Neonの無料枠には直近履歴（通常24時間程度）を巻き戻せる機能もありますが、これは短時間のうっかりミス救済用です。長期保全は上記の日次GitHub Actionsバックアップが本体です。

## よくある質問

**Q. サイトにアクセスできない**  
A. Renderのサーバーがスリープしている場合があります。30〜50秒待ってからリロードしてください。それでもだめな場合はRenderのダッシュボードでサービス状況を確認してください。

**Q. 住民からパスワードを忘れたと言われた**  
A. 「👥 管理者設定」タブから該当の部屋番号のパスワードをリセットできます。リセット後に仮パスワードを住民に伝えてください。

**Q. 蔵書データはどこから来ているか**  
A. Librarylifeというサービスと同期しています。蔵書の増減はLibrarylife側の登録に連動します。アプリ側で直接書籍を追加・削除することはしないでください。

**Q. お知らせが住民に届いているか確認したい**  
A. お知らせを投稿すると住民のタブに未読バッジが表示されます。投稿後に住民側の画面を確認してみてください。
